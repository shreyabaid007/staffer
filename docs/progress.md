## Works end-to-end right now
- `uv run dsm match --role-id <ID>` — runs the **full 9-step query pipeline** for a single role parsed from the demand CSV: (1) load config, (2) parse demand CSV → typed `OpenRole`, (3) check freshness (AD-087), (4) load index + gate pre-filter (location AD-086 + availability), (5) exact hard-skill filter (AD-088), (6) hybrid recall (dense + BM25 + RRF — passthrough when `index.recall.enabled=false`, AD-089), (7) cross-encoder rerank with top-k truncation, (8) hydrate serving `Candidate`s via `CandidateStore` (AD-091), (9) DSPy clarify → LLM sub-scores (skill match + availability + seniority + feedback) → deterministic combine + adjacency map + cited flags → rank → `ShortlistResult` / `NoMatchResult` JSON. Freshness warnings surface as `FRESHNESS_WARNING` flags (AD-092). Seniority is a soft signal, never a gate (AD-090). LLM sub-scores clamped to [0,1] before combine.
- `uv run dsm match --query "<prose>"` — **natural-language front door** (c-006, AD-110/111): a single-shot DSPy parse over `PseudonymisedLM` (pass-through) turns prose into the **same** typed `OpenRole` the CSV produces, echoes it for confirmation (`--yes` to skip), then runs the identical spine. Relative dates are LLM-resolved (today injected) + **Python-validated** before the availability gate; `co_location_required` is **Python-derived** (the LLM never sets a gate, AD-002); a missing location/start triggers **one** bounded Python clarification (no LLM loop). Exactly one of `--query`/`--role-id` required.
- `uv run dsm serve` — **web frontend** (c-008, AD-113): boots a FastAPI JSON API + single static page at `http://127.0.0.1:8000` (host/port from `config/default.yaml::web.*`, CLI `--host`/`--port` override). Two front doors over the **same** spine: NL intake (`POST /intake` → echo + confirm → `POST /match/query` → shortlist) and demand-CSV upload (`POST /demand/parse` → `POST /match/role`). Resume PDF via `GET /resume/{candidate_id}`, append-only decision capture via `POST /decisions`, health at `GET /healthz`. The match response is a web-local view pairing the pseudonymous `candidate_id` with the de-anonymised identity — frozen `dsm/models.py` untouched, `dsm/web` never imports `dsm.pii`. All LLM calls route through the TokenOps proxy for cost tracking (when `tokenops.enabled=true`).
- `uv run dsm explain --role-id <ID>` — re-runs the same pipeline and dumps full lineage per §9: role parse, freshness verdict, gate/filter exclusions, recall/rerank trace, per-candidate score breakdown, and final ranking.
- `uv run dsm ingest` — lands raw files (`data/raw` → bronze), parses (CSV/PDF/markdown) → bronze, normalizes → silver `NormalizedRecord`s, then **enriches resumes/feedback through the PII boundary (redact → leak-scan → DSPy extract @ temp 0 → de-anonymize → verify-quote) and merges silver+enriched into one canonical `GoldCandidate` per `candidate_id`** (§7 authority, resume↔feedback conflicts recorded never averaged), reconciles against prior gold (tombstones departed; an idempotent re-run is a no-op for gold), writes `gold/<cid>.json` (+`gold_hash`), and prints PII-safe `── Silver ──` + `── Gold ──` summaries; exits 1 on errors / leak-block. `make smoke` runs an opt-in real-data smoke.
- `uv run dsm index` — builds the **write-time** index from gold: projects each `GoldCandidate` into a PII-free `CandidateIndexRecord` (filter fields from `Sourced[...].value`), builds a deterministic capability-only `embed_text` + a `skill_set` that **excludes feedback-denied skills** (`demonstrated is False`, AD-081), embeds the passage via the injected `EmbedClient` (`ModalEmbedClient` in prod, `FakeEmbedClient` in tests), and upserts into the Milvus Lite `candidates` collection (`data/index/milvus.db`: PK `candidate_id`, dense `FLOAT_VECTOR` dim 768 / metric IP, `skill_set` `ARRAY<VARCHAR>` + BM25 `SPARSE_FLOAT_VECTOR`). Re-embeds + upserts only when stored `(gold_hash, model_version)` doesn't match (AD-082); tombstones delete, thin profiles (missing grade/location/availability) are skipped. Prints a PII-safe `── Index ──` summary (indexed / skipped-unchanged / tombstoned-removed / thin-skipped). `embed_text`/`skill_set` are PII-free **by construction** — no outbound known-PII scan at index time (AD-084). Index schema carries `remote_within_country: bool` + `onsite_cities: ARRAY<VARCHAR>` (AD-086).
